## 全局位姿修正

原始 `poses.txt` 实际记录 `base_link → head_camera`，不能直接作为移动机器人的世界
位姿。过滤脚本按相机时间戳分别插值里程计和头部相机位姿，并计算：

```text
odom_T_camera = odom_T_base_link @ base_link_T_head_camera
```

生成的全局相机平移从约 `[0.063, 0.021, 1.512] m` 变化到
`[6.062, -1.347, 1.493] m`，累计轨迹长度约 14.05 m。数据加载器也已改为优先
读取 `tick_index.json` 中的真实时间戳，过滤后仍保留 157.81 s 时间跨度，最大帧间隔
为 0.332 s，不会把跳帧错误压缩为固定 10 Hz。

### 供外部模块调用的查询 REST API

服务启动时预加载图和 Sentence-T5 模型，默认只监听本机回环地址：

```bash
source /opt/ros/jazzy/setup.bash
source .repro/ros2_ws/install/setup.bash
source .repro/venv/bin/activate
export DAAAM_KEY='...'  # 仅 /v1/query/ask 需要；纯检索可不设置
python scripts/serve_query_api.py \
  --dsg output/g1_20260713_170500_sync10ms_full/out_20260714_114803/dsg_updated.json \
  --host 127.0.0.1 --port 8765
```

OpenAPI 文档位于 `http://127.0.0.1:8765/docs`。接口如下：

```bash
# 纯本地语义检索：不会调用任何外部 API
curl -sS http://127.0.0.1:8765/v1/query/retrieve \
  -H 'Content-Type: application/json' \
  -d '{"query":"white ceiling light","top_k":5}'

# 检索后受图证据约束的模型问答：密钥始终只在服务端
curl -sS http://127.0.0.1:8765/v1/query/ask \
  -H 'Content-Type: application/json' \
  -d '{"query":"白色天花板灯在哪里？","top_k":5}'
```

`/v1/query/retrieve` 返回候选的相似度、节点 ID、描述、三维位置和观测时间。
`/v1/query/ask` 会先将问题转为英文检索短语，再由 `qwen3.7-plus` 仅依据候选节点回答，
并在回答中引用节点 ID。服务缺少 `DAAAM_KEY` 时，后者返回 HTTP 503；不影响纯检索接口。
如需跨机器访问，显式指定 `--host 0.0.0.0` 并在网络层限制访问范围。
