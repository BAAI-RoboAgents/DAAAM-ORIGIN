# 实时动态语义地图改造路线

## 当前基线

G1 `20260713_170500` 的 789 帧最终数据已完成 FoundationStereo、RGB-D
局部约束、几何闭环、全局位姿图、时序深度过滤和 Hydra 实跑。

| 模块 | 本次实测 | 约等效频率 |
|---|---:|---:|
| FoundationStereo，32 iterations | 1.240 s/帧 | 0.81 Hz |
| DAAAM CV 前端 | 0.228 s/帧 | 4.38 Hz |
| Hydra 融合 | 0.104 s/帧 | 9.62 Hz |
| CV + Hydra 当前串行路径 | 0.332 s/帧 | 3.01 Hz |

CV 前端的主要耗时为：track 后处理 106.6 ms、分割 58.3 ms、跟踪
40.6 ms、输出标签图 21.9 ms。完整 789 帧任务约 5 分 27 秒，语义 worker
在主循环之后仍需排空队列。

最终 Hydra 后端图包含 594 个节点和 1106 条边，Mesh 为 81,607 个顶点和
106,047 个三角面。空间轮廓已可辨认，但存在 4,013 个连通分量，最大分量仅占
1.84% 的三角面。当前系统因此是可复现的离线基线，还不是实时动态地图。

## 目标定义

不要让所有模块都追随相机帧率。建议使用四条不同频率的更新路径：

| 路径 | 建议频率 | 延迟目标 | 内容 |
|---|---:|---:|---|
| 实时状态估计 | 15 到 30 Hz | P95 小于 50 ms | VIO、轮速/IMU、连续 pose |
| 局部几何与动态跟踪 | 10 Hz | P95 小于 100 ms | 深度、局部 TSDF、动态实例 |
| 语义更新 | 1 到 2 Hz，事件触发 | P95 小于 2 s | 分割、ReID、开放词汇描述 |
| 全局后端 | 0.2 到 1 Hz | 不阻塞前端 | 闭环、pose graph、地图形变 |

关键验收条件：输入队列有界；过载时按绝对时间和内容价值丢帧；前端不会等待
DAM 或全局优化；所有异步结果都通过 `sensor_time_ns` 和稳定实体 ID 回写。

## P0 必须改造

### 1. 在线传感器与时间契约

- 将当前文件式预处理变为 ROS 2 或等价流式节点，缓存左右目、pose、IMU 和轮速。
- 使用硬件时间或统一时钟域，在线做双目配对和 pose 插值。
- 消息贯穿 `sensor_time_ns`、源序号、校准版本和处理版本，禁止按队列序号配 pose。
- 加入迟到、乱序、时钟跳变和左右目超时策略，并导出可观测指标。

### 2. 实时深度后端

- FoundationStereo 当前 0.81 Hz，是首要瓶颈。先基准测试更低迭代数、FP16、较低
  分辨率、`torch.compile` 或 TensorRT，再决定是否保留为实时后端。
- 将深度推理放在独立进程和 CUDA stream 中，使用固定显存池、预分配张量与 pinned
  memory，避免每帧加载和同步。
- 输出逐像素置信度、左右一致性和遮挡掩码；TSDF 融合权重必须使用置信度，而不是
  将所有非零深度等权累计。
- 保留高质量 FoundationStereo 作为关键帧或后台修正后端，同时准备低延迟深度模型
  或硬件深度作为实时前端。

### 3. 状态估计与局部地图

- 当前离线 RGB-D 图优化不能直接搬到逐帧前端。增加 IMU/轮速紧耦合 VIO 或成熟的
  stereo-inertial SLAM，机器人 odom 继续作为先验。
- 前端维护固定时长滑窗并边缘化旧状态；局部 RGB-D 约束增量求解，不重复优化全轨迹。
- 闭环采用地点检索、几何验证、重力过滤三级门禁，在后台增量优化。
- 全局 pose 变化后使用 deformation graph 或 submap 变换修正地图，避免清空重建。

### 4. 静态与动态分层

- 分割结果先生成动态候选，人、可动物体和不满足多视图静态一致性的像素不得进入
  永久静态 TSDF。
- 为动态实例维护 `position/velocity/covariance/last_seen_ns`，按真实时间做预测和关联。
- 同一实例区分“当前状态”和“历史轨迹”；对象离开后从实时占据层移除，但保留事件。
- 只有经过时间稳定性确认的表面才能从临时层晋升到静态地图。

### 5. 有界异步调度

- 将采集、深度、分割跟踪、局部融合、语义描述和全局优化拆成有界队列。
- 每条队列定义容量、截止时间和丢弃策略。优先保留 pose 运动、视觉事件、闭环候选和
  watchdog 帧，严格重复帧最先丢弃。
- DAM-3B 不得占住实时前端 GPU。采用独立 GPU、显存配额或调度时间片；积压时合并
  同一 track 的请求。
- 为每帧记录 queue wait、GPU time、CPU time、端到端延迟和丢帧原因。

## P1 质量与语义闭环

- 优化 `process_tracks`：向量化 mask 深度统计、避免重复 polygon/mask 转换，并缓存
  不变的 ReID 与几何特征。
- 减少整幅 `label_image` 的重复构造和复制，让 Hydra 接收紧凑实例掩码或共享内存。
- 语义描述按新实体、低置信度、内容变化和用户查询触发，不按固定 90 帧整批触发。
- 当前运行产生 458 条语义修正，但统计仍显示 0 条 applied，且有 1149 条 pending。
  需要实现基于稳定实体 ID 的回写确认、幂等更新和失败重试。
- 维护跨闭环和跨会话的对象 ReID；合并前同时检查外观、3D 位置、尺度和时间可达性。
- 小连通 Mesh 分量按面积、观测次数和生命周期清理；TSDF block 使用活动窗口和 LRU
  换出，控制显存与内存上界。

## P2 工程化

- 增加在线回放基准，分别模拟 5/10/15 Hz、动态人员、停留观察、网络延迟和 GPU
  降频，禁止仅用离线总耗时判断“实时”。
- 设定回归门禁：pose ATE/RPE、深度时序一致率、动态实例 MOTA/ID switch、静态地图
  重影、Mesh 连通性、语义覆盖率、P50/P95/P99 延迟和峰值显存。
- 增加进程健康检查、队列积压告警、模型降级、自动重启和中间状态 checkpoint。
- 将校准、模型、阈值、Git/Submodule SHA 和硬件信息写入每次运行清单。

## 推荐里程碑

1. **实时几何 MVP**：不运行 DAM，双目深度、VIO、动态掩码和局部 TSDF 稳定达到
   10 Hz，连续运行 30 分钟无无界积压。
2. **异步语义 MVP**：语义以 1 Hz 或事件触发更新，前端 P95 不因 DAM 增加超过
   10 ms，描述能可靠回写到对象 ID。
3. **动态地图 MVP**：动态对象不污染静态 TSDF，速度和生命周期可查询，遮挡后能
   重关联。
4. **全局一致版本**：几何闭环在后台生效，submap 和对象随 pose graph 一致形变，
   前端不中断。
5. **部署版本**：完成长时压力、故障注入、资源上限和质量门禁，再考虑多机器人或
   跨会话地图。
